# MovieMVC

Уявімо, що ми створюємо сайт-каталог фільмів
Для тестового завдання ми будемо працювати лише з двома повʼязаними сутностями:
фільмом і категорією
В BD це відображається наступними таблицями:
Table “films”
Int Id auto increment primary key
Varchar 200 name
Varchar 200 director
Datetime release
Table “categories”
Int Id auto increment primary key
Varchar 200 name
Int nullable parent_category_id key
Table “film_categories”
Int Id auto increment primary key
Int film_id key
Int category_id key
Нові таблиці в BD ми не створюємо.
Потрібно реалізувати бєкенд і фронтенд частину що буде виконувати наступні функції:
1) CRUD операції для фільму за допомогою asp.net mvc
2) JS-class який дозволяє переглянути повʼязані категорії і додати/прибрати нові з
можливістю мультивибору на сторонці перегляду інформації про окремий фільм.
А також asp.net web api для цього класу (для самого інпуту селекту можна
використовувати готову бібліотеку)
3) CRUD операції для категорії за допомогою asp.net mvc (заборонено замикати
кільце з материнських категорій)
4) Сторінка з переліком всіх фільмів таблицею з додатковими стовпчиком переліку
категорій фільму через кому (бонусом використати бібліотеку що додасть до
таблиці сортування за датою і фільтрацію за режисером, *великим бонусом
реалізувати фільтр за категорією)
5) Сторінка з переліком всіх категорій таблицею, додати два стовпчики: кількість
фільмів у категорії і рівень вкладеності категорії
Для дизайну достатньо використати базовий bootstrap
